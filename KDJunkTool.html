KDJunkTool

ParameterTypeDescriptionDefault value
junk_tool_luggedbooleanwhether to slow down the player while in inventorytrue
junk_tool_dropbooleanwhether to drop the object upon successful usefalse

Unlike other inventory objects, tools (objects that can be used to frob other objects) cannot be given the Junk type. KDJunkTool replaces much of this behavior for a tool object. While the object is in inventory:

  • No other inventory item or weapon can be accessed.
  • Objects in the world cannot be frobbed, except to be picked up. (The latter is a technical limitation.)
  • The object is kept at the center of the screen (always in use).
  • The object still functions as a tool, interacting with any target object it is used to frob.

The object should have the following settings in the Engine Features\FrobInfo property:

World Action
Move (To place the object in inventory when frobbed in the world.)
Inv Action
  • Script (No action), or
  • Move (To throw the object when it is frobbed without a target object.)
Tool Action
Script (Combined with a script, such as StdKey or LockPick, or an A/R stimulus.)

If the junk_tool_lugged parameter is true, the player will be slowed down to 60% of normal speed while holding the object, and the grunting schemas garlift and gardrop will be played when the object is picked up and put down, respectively. This replicates the behavior of the Lugged script (which should not be used with this one), except that this script handles slaying of the object correctly. If this parameter is used, the object's mass should be increased (in the Physics\Attributes property) for realistic throwing/dropping physics.

For key objects (having the StdKey script), a successful use results in the key being slain. This allows various responses to successful use:

remain in inventory
no special setup
be destroyed
set Game\Damage Model\Slay Result property to Normal
drop to ground
set junk_tool_drop parameter to true
